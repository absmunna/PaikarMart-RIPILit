- frontend/
- backend/
- api/
- uiux/

1. Folder structure correctness (modular separation)
2. Duplicate files/components
3. Broken imports / unused files
4. Inconsistent naming
5. Missing environment variables
6. Type errors (TypeScript / Zod)

8. Config usage correctness (feature flags, payment, delivery)

Compare current system with requirements:

- Multi-seller system

Identify:

- Fake data usage (e.g., SellerContext local state)

- Dead routes / unused endpoints

Classify:

- 🔴 Critical (must fix now)
- 🟡 Medium
- 🟢 Low

Fix ONLY critical issues that may:

- Break future development
- Cause data inconsistency
- Cause security risks

- Broken API route → fix

- Refactor entire system
- Start new features
- Change database structure (Phase 1 will do that)
